# KRC AI 디지털 혁신 공모전 — 프로젝트 작업 공간 규칙

## 범위
- 이 규칙은 이 프로젝트 폴더 안에서만 적용된다. 다른 프로젝트나 과거 세션의 규칙은 참조하지 않는다.
- 데이터는 오직 `base_data/`와 `공공데이터 활용대상/` 안의 파일만 사용한다. 존재하지 않는 데이터나 수치를 지어내지 않는다.

## 팀 구성
- 모든 작업은 `.agents/agents.md`에 정의된 5개 페르소나(A~E) 중 하나로 수행한다.
- 각 페르소나는 반드시 `.agents/skills/` 안의 해당 업무 매뉴얼을 먼저 읽고 그 순서를 따른다.

## 산출물 원칙
- 모든 중간 산출물은 반드시 `production_artifacts/` 폴더에 파일로 저장한다 (채팅창에만 답하고 끝내지 않는다).
- PoC 코드는 트랙에 맞는 폴더에 저장한다: 인터랙티브 데모는 `app_build_streamlit/`, 정적 대시보드는 `app_build_static/`, 소개 페이지는 `landing_page/`. 정적 대시보드용 데이터 전처리 스크립트는 `analysis/`에 저장한다.
- 각 Phase가 끝나면 `progress_checklist.md`의 해당 항목에 체크하고 점수를 기록한다.

## 위험 명령 승인 정책 (Agent Security Mode가 Full Access여도 항상 적용)
IDE의 터미널 자동 실행 설정(Always Proceed)과 무관하게, 아래 행동은 반드시 실행 전 채팅으로 사용자에게 먼저 확인을 구하고, 사용자가 "예" 또는 "승인"이라고 답한 뒤에만 실행한다:
- 파일/폴더 삭제 (`rm`, `rmdir` 등) 또는 기존 최종 산출물(`production_artifacts/04_proposal_final.md` 등) 덮어쓰기
- `git push`, `git push --force`, 원격 저장소 관련 모든 명령
- `base_data/`, `공공데이터 활용대상/` 등 원본 데이터 폴더 내 파일 수정
- 이 목록에 없는 일반 작업(패키지 설치, 코드 실행, 로컬 파일 생성 등)은 자동으로 진행해도 된다.

## 배포 원칙 (100% 사람이 직접 수행, 에이전트 자동화 대상 아님)
- Netlify 배포(수동 배포, CLI, 드래그앤드롭 등)와 Streamlit Community Cloud 연결은 에이전트가 절대 스스로 실행하지 않는다.
- 에이전트의 역할은 배포에 필요한 파일(정적 사이트, requirements.txt 등)을 준비하고 GitHub 커밋까지만이며, `git push`와 실제 배포 버튼/명령은 항상 사람이 직접 수행한다.
- 배포 관련 요청을 받으면 에이전트는 실행 대신 5장의 수동 배포 절차를 안내한다.

## 언어
- 모든 산출물은 한국어로 작성한다. 공식 기획서 톤을 유지한다.
